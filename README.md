A full-stack flashcard app with a minimalist, dark UI. Next.js frontend, a Go backend speaking gRPC/Connect, Postgres for storage, and Google sign-in.

## Features

- **Folders → decks → cards** — organize study material with full create / rename / delete.
- **Review mode** — a 3D flip card with keyboard shortcuts and live progress (current card, reviewed, remaining).
- **Bulk JSON import** — paste an LLM-generated deck and create it (and all its cards) in one atomic operation, with a copyable prompt to feed any model.
- **Google sign-in** — OAuth2 with secure, signed-JWT session cookies.
- **Considered motion** — entrance/scroll reveals, press feedback, and `prefers-reduced-motion` support throughout.

## Tech stack

| Layer       | Tools |
|-------------|-------|
| Frontend    | Next.js (App Router), TypeScript, Tailwind CSS v4, Motion, Connect-Web, Phosphor Icons |
| Backend     | Go, gRPC + [Connect](https://connectrpc.com), pgx, [sqlc](https://sqlc.dev) |
| Database    | PostgreSQL ([Neon](https://neon.tech)) |
| Auth        | Google OAuth2, JWT sessions |
| Codegen/DX  | [buf](https://buf.build), golang-migrate, pnpm |

## Architecture

```
 browser ──Connect/HTTPS──▶ Go HTTP server ──┐
 (Next.js on Vercel)        (Connect + OAuth) ├─▶ services ──pgx/sqlc──▶ Postgres (Neon)
                            gRPC :50051 ──────┘
```

The domain logic (folders, decks, flashcards, import) is written once and exposed over both
native gRPC and the browser-friendly Connect protocol. All queries are generated by sqlc
(parameterized — no string interpolation) and every request is validated at the API boundary.

## Getting started

### Prerequisites

- **Go** 1.24+
- **Node** 20+ and **pnpm**
- A **Postgres** database (a free [Neon](https://neon.tech) project works great)
- Optional, only for regen/migrations: [`buf`](https://buf.build), [`sqlc`](https://sqlc.dev), [`golang-migrate`](https://github.com/golang-migrate/migrate)

### 1. Clone & configure

```bash
git clone https://github.com/Shivam583-hue/trueflashcard.git
cd trueflashcard

cp server/.env.example server/.env   # set DATABASE_URL, JWT_SECRET, GOOGLE_* …
cp web/.env.example   web/.env       # defaults point at localhost:8080
```

For Google sign-in, create an OAuth client in the Google Cloud Console and set
`GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`, with `http://localhost:8080/auth/google/callback`
as an authorized redirect URI.

### 2. Install dependencies

```bash
make install
```

### 3. Run the database migrations

```bash
make migrate
```

### 4. Start everything

```bash
make run
```

- Frontend → http://localhost:3000
- Backend → http://localhost:8080 (Connect API + auth), `:50051` (gRPC)

`Ctrl-C` stops both.

```

├── web/                  # Next.js frontend
│   └── src/{app,components,gen,lib}
├── server/               # Go backend
│   ├── cmd/server/       # entrypoint
│   ├── proto/            # protobuf service definitions
```

## Commands

| Command         | Description |
|-----------------|-------------|
| `make run`      | Start backend and frontend together |
| `make install`  | Install frontend and backend dependencies |
| `make build`    | Build both (`go build` + `pnpm build`) |
| `make test`     | Run backend tests |
| `make migrate`  | Apply database migrations |
| `make proto`    | Regenerate gRPC/Connect code (Go + TypeScript) |
| `make help`     | List all targets |

## Deployment

The frontend deploys to **Vercel** (root directory `web`). The backend is a single static
binary that runs anywhere — the included `deploy/` kit covers a **VPS exposed via Cloudflare
Tunnel** (systemd unit, `cloudflared` config, env template, and step-by-step instructions).
See [`deploy/README.md`](deploy/README.md).
